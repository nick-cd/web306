The Request/Response Process

  1. The client sends a request to the server.
  2. In response, the server processes a server-side language like PHP, Python or Ruby on the back end and returns HTML, CSS and JavaScript code to display a web page on the front end.

XAMPP

  1. XAMPP is a collection of software which allows us to run server and database software in our local environment
  2. XAMPP stands for X (your operating system), Apache, MySQL/MariaDB, PHP
  3. This forms what is known as your development "stack"
  4. The LAMP Stack
    1. The traditional "stack" which is used by developers is known as the LAMP stack
    2. The LAMP Stack constists of:
      1. Linux is used on servers but XAMPP allows us to use Windows or Mac OS as well
      2. Apache is the server software
      3. MySQL or MariaDB which are database software
      4. PHP is our back-end scripting language
    3. If an employer asks for LAMP developers or asks you what "your stack" is, this is what they are referring to
    4. The second "P" in XAMPP actually stands for Perl, but we won't be using that
    5. Python developers also call their stack a LAMP stack
    6. Other stacks include the LAMR stack (Ruby developers) and the MEAN stack (NodeJS developers)
  5. Download XAMPP at the Apache Friends website (they'll be there for you!)
  6. Once you have XAMPP installed turn on Apache and MySQL/MariaDB
  7. Remember to save all projects which use technologies that are dependant on server software such as PHP and AJAX in C:/xampp/htdocs so that the server can find them, unless you have set up a virtual host (ask Tim about this in the lab)
